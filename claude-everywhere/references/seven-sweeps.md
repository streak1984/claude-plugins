# Seven Sweeps — Flerpassredigering

Sekvensielle redigeringspass for innholdsskills. Hvert pass har ett fokus. Simultane sjekker overser feil som sekvensielle fokuserte pass fanger opp.

Kjør alle syv i rekkefølge på det ferdige utkastet. Fiks problemer stille — ikke list dem.

---

## Pass 1: Klarhet

Kan leseren forstå hver setning ved første lesing?

- Uklar pronomenreferanse ("den", "det", "dette" uten tydelig referent)
- Begravde poeng — hovedpoenget dukker opp midt i eller på slutten av avsnittet
- Fagsjargong uten kontekst (forklar eller erstatt)
- Setninger som krever to lesinger — skriv om
- Doble nektelser ("ikke uvanlig" → "vanlig")

## Pass 2: Stemme og tone

Er stemmen konsistent gjennom hele teksten?

- Formelt/uformelt-skift midt i teksten
- Personlighetsinkonsekvens (saklig → plutselig muntlig, eller omvendt)
- Les høyt — hører du en person eller en mal?
- Passer tonen til formatet? (nyhetsbrev er personlig, artikkel er faglig-men-tilgjengelig)
- Stiv formulering som kan løses opp uten å miste presisjon

## Pass 3: "Så hva?"

Svarer hvert utsagn på hvorfor leseren bør bry seg?

- Påstander uten konsekvens for leseren → legg til "det betyr at..."
- Egenskaper uten fordeler → oversett til verdi
- Fakta som bare informerer, ikke overbeviser → koble til leserens situasjon
- Avsnitt som "setter scenen" uten å komme til poenget → kutt eller omskriv
- Hvis du fjerner avsnittet og ingenting mangler — kutt det

## Pass 4: "Bevis det"

Er hver påstand understøttet?

- Uunderbyggede påstander ("mange bedrifter opplever...", "det viser seg at...") → legg til tall, kilde, eksempel
- Manglende sosial bevisføring → spesifikke tall, navn, case
- Ufortjente superlativer ("best", "størst", "revolusjonerende") → fjern eller underbygg
- Vage referanser ("studier viser", "eksperter mener") → hvem? hvilke? når?
- Spesifikt > generisk: "143 norske bedrifter" > "mange bedrifter"

## Pass 5: Spesifisitet

Er teksten konkret?

| Vagt | Spesifikt |
|------|-----------|
| "spar tid" | "spar 4 timer i uken" |
| "øk konvertering" | "øk konvertering fra 2,1% til 3,8%" |
| "mange kunder" | "1 200 kunder i 14 land" |
| "nylig" | "i januar 2025" |
| "betydelig vekst" | "47% vekst på 6 måneder" |
| "raskere" | "50ms responstid, ned fra 400ms" |

- Runde tall → eksakte tall der mulig
- Vage verb (forbedre, effektivisere, optimalisere) → beskriv hva som faktisk skjer
- "Ting" og "dette" → navngi det spesifikt

## Pass 6: Følelse

Får teksten leseren til å føle noe?

- Flat informativ tekst → legg til en levende før-tilstand ("Du sitter med 14 faner åpne, tre halvferdige dokumenter...")
- Abstrakte fordeler → sensoriske detaljer og konkrete situasjoner
- Manglende menneskelig element → korte mikrohistorier, gjenkjennbare situasjoner
- Refleksjonsspørsmål som inviterer leseren til å kjenne seg igjen
- Ikke manipulasjon — ekte forbindelse med leserens virkelighet

## Pass 7: Null risiko

Er hindringene for handling fjernet?

- Friksjon nær CTA-er (forvirrende neste steg, uklart hva som skjer etter klikk)
- Ubesvarte innvendinger ("men hva koster det?", "fungerer det for oss?")
- Manglende tillitssignaler (garantier, referanser, resultater)
- Risikoreversering: "ingen forpliktelser", "gratis prøveperiode", "pengene tilbake"
- Siste sjanse: ville DU klikket? Hvis ikke — hva mangler?

|-------|-------------|
| utilize / utnytte | bruke |

| streamline | forenkle |

## Setnings- og avsnittssjekk

- **En idé per setning.** Hvis setningen har "og" + nytt poeng, del den.
- **Variér lengde.** Kort. Så lengre. Så kort igjen. Monoton rytme bedøver leseren.
- **Front-load det viktige.** Poenget først, kontekst etterpå.
- **Maks 25 ord per setning** som tommelfingerregel. Lange setninger kan fungere — men sjelden mer enn én per avsnitt.
- **2-4 setninger per avsnitt** for web/e-post. Ett-setnings-avsnitt for punch. Aldri mer enn 5 setninger.
- **Kutt "det"-åpninger.** "Det er viktig å..." → "Du må..." / "Husk at..."
