# Article Structural Template

Use this skeleton when generating articles in campaign mode. Fill in placeholders with fresh copy based on the brief and client profile.

## Structure

```yaml
---
title: "{title — under 60 chars, contains primary keyword}"
description: "{meta description — 150-160 chars, ad copy style}"
slug: "{url-slug — short, keyword-rich}"
date: YYYY-MM-DD
keywords:
  - "{primary keyword}"
  - "{secondary keyword 1}"
  - "{secondary keyword 2}"
  - "{secondary keyword 3}"
author: "{from client profile}"
heroImage: ""
---
```

## Structural Approaches

Choose ONE structure based on the brief's angle. The template is a starting point — if the reader can predict the next section, restructure.

### Option A: Problem-Solution
Hook intro → Problem (with data) → Solution (concrete, actionable) → Evidence (cases, numbers) → Neste steg (CTA)

### Option B: Narrative/Case-Driven
Concrete story or case → Extract the principle → Apply to reader's situation → Supporting evidence → Neste steg (CTA)

### Option C: Contrarian Argument
Bold claim that challenges conventional wisdom → Why the common approach fails → The alternative (with proof) → How to implement → Neste steg (CTA)

## Section Guidelines

- **Hook intro**: 2-3 sentences. First sentence earns the second. No definitions, no "I dagens...".
- **Body sections**: 2-4 sections, each 150-400 words. Front-load the point in each section.
- **Neste steg**: CTA tied to the core message. 2-3 sentences. Specific action, benefit-driven.

## Variation Rules

- **Vary paragraph length.** One-sentence paragraphs are powerful. Use them.
- **Structure should be invisible to the reader.** If it feels like a template, restructure.
- **Headings tell the story.** A reader skimming only headings should get the argument.
- **No fixed section count.** 3 sections or 6 sections — whatever the content needs.

## Total target: 800-1200 words
